# Contributing

Thanks for contributing to HandLock Care Center.

## Scope

Useful contributions include:

- bug fixes
- documentation improvements
- WordPress compatibility updates
- PHP compatibility fixes
- code cleanup and refactoring
- safer defaults for open-source reuse

## Before You Open a Pull Request

1. Keep the change focused.
2. Do not commit secrets, customer data, SQL dumps, backups, or treatment media.
3. Explain any schema, environment, or migration impact clearly.
4. Prefer preserving backward compatibility unless the change is intentionally breaking.

## Development Notes

- This project is a WordPress plugin.
- PHP source lives mainly under `includes/`.
- Frontend and admin assets live under `assets/`.
- The repository is a sanitized open-source snapshot, not a full production environment export.

## Pull Request Checklist

- code is relevant to the repository
- no private data or environment-specific credentials are included
- documentation is updated when behavior changes
- new assumptions or limitations are stated in the PR description
